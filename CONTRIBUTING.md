## How to Contribute

Thank you for helping grow Digging Code! Your contributions make a real difference.

### 1. Fork the Repository

Click the **Fork** button at the top-right of this repository to create your own copy.

### 2. Create a New Branch

Create a branch for your contribution to keep changes organized:

```bash
git checkout -b add-package-[package-name]
```

Replace [package-name] with the name of the package you are adding.

### 3. Add Your Package

Add your package to the table in packages.md. Each entry should include:

| Package Name | Description | Stars | Usage / Notes |
|-------------|-------------|-------------|---------------|
| ExamplePackage | A brief description of the package | ‚≠ê 5K | Short usage example or tip |

- Package Name ‚Äì the name of the library or tool
- Description ‚Äì a concise one-line explanation
- Stars ‚Äì popularity/quality indicator
- Usage / Notes ‚Äì optional usage tip or example

### 4. Commit Your Changes

```bash
git add packages.md
git commit -m "Add [package-name] package"
```

### 5. Push Your Branch

```bash
git push origin add-package-[package-name]
```

### 6. Open a Pull Request

- Go to your fork on GitHub.
- Click **Compare & pull request**.
- Target the `main` branch of this repository.
- Add a short description of the package and its purpose.
- Submit the PR for review.
